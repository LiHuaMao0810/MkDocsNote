# 面经总结

本模块汇总整理大模型面试中的高频问题、常见考点和答题技巧。

## 内容分类

### 基础理论
收集关于 Transformer、注意力机制、位置编码等基础问题的面经。

### 模型对比
整理常见的模型对比问题，如 BERT vs GPT、扩散模型 vs GAN 等。

### 工程实践
汇总训练优化、部署加速、资源管理等工程问题的答题要点。

### 前沿技术
记录关于最新技术和研究方向的讨论，如 MoE、长上下文、多模态等。

### 手撕代码
整理常见的代码实现题，如：
- 实现多头注意力
- 实现 RoPE 位置编码
- 实现简单的 Transformer 层
- 实现 Top-K 采样

### 场景题
收集实际应用场景的问题，如：
- 如何设计一个对话系统？
- 如何优化模型的推理速度？
- 如何处理长文本输入？
- 如何评估模型性能？

## 面试技巧

### 回答策略
1. **先总后分**：先给出简洁的核心答案，再展开细节
2. **结合实例**：用具体模型或论文举例说明
3. **对比分析**：说明不同方案的优缺点和适用场景
4. **承上启下**：适时引导话题到自己熟悉的领域

### 常见问题类型
- **概念解释**：解释清楚原理和作用
- **方案设计**：考虑完整性和可行性
- **问题排查**：展示系统性的分析思路
- **性能优化**：从多个维度提出优化方案

### 准备建议
1. 梳理自己的项目经历，准备项目介绍
2. 总结每个项目的技术难点和解决方案
3. 了解目标公司的技术栈和产品
4. 准备几个高质量的问题问面试官

## 企业分类

### 互联网大厂
- 重视基础理论和系统设计
- 注重工程实践能力
- 可能有手撕代码环节

### AI 公司
- 关注前沿技术和论文理解
- 注重创新能力和研究潜力
- 可能深入讨论具体模型细节

### 传统企业
- 重视实际落地能力
- 关注成本和效率
- 可能更关注应用场景
